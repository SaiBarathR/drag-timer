# Drag Timer 1.3.1

## Highlights

Drag Timer 1.3.1 is the public-release hardening update. It introduces a universal Apple Silicon and Intel application, a real macOS icon, explicit MIT licensing, and a release pipeline that cannot publish until the source, tests, bundle, archive, signature, version, and checksum have all passed validation.

## Changes

- Package one native executable containing both `arm64` and `x86_64` slices.
- Add an original Drag Timer application icon to Finder and other macOS surfaces.
- Gate releases on `swift build`, the XCTest suite, and the deterministic self-test.
- Require the release tag, bundle version, positive build number, and tracked notes to agree.
- Verify the final ZIP by extracting it into a clean directory and rechecking its checksum, version, architectures, icon, and ad-hoc signature.
- Refuse to overwrite an existing release or release asset.
- Add an MIT license and complete installation, verification, architecture, and release documentation.

## Compatibility

- Requires macOS 14 or newer.
- Runs natively on Apple Silicon and Intel Macs (`arm64 + x86_64`).
- This update changes packaging and documentation only; persisted timer, preset, routine, settings, expiry, and history formats are unchanged from v1.3.0.

## Installation

1. Download `Drag-Timer-1.3.1-macos-universal.zip` and `SHA256SUMS.txt` from this release.
2. In Terminal, change to the download directory and run `shasum -a 256 -c SHA256SUMS.txt`.
3. Unzip the archive and move `Drag Timer.app` to Applications.
4. Control-click the app, choose **Open**, and confirm. You can also use **Open Anyway** in System Settings > Privacy & Security.

## Verification

After extraction, verify the universal executable and bundle signature:

```sh
lipo -archs "Drag Timer.app/Contents/MacOS/DragTimer"
codesign --verify --deep --strict --verbose=2 "Drag Timer.app"
/usr/libexec/PlistBuddy -c 'Print :CFBundleShortVersionString' "Drag Timer.app/Contents/Info.plist"
```

The commands should report both `arm64` and `x86_64`, a valid signature, and version `1.3.1`.

## Known limitations

The app is ad-hoc signed, not Developer ID signed or notarized. macOS requires a one-time Gatekeeper confirmation. Intel compilation and archive structure are verified automatically; the tracked release checklist records whether the exact release candidate has also been exercised on physical Intel hardware.

## Checksums

`SHA256SUMS.txt` is generated from the final ZIP after packaging. Keep it beside the downloaded ZIP and run `shasum -a 256 -c SHA256SUMS.txt` before opening the app.
