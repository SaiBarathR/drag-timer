# Drag Timer 1.3.0

## Highlights

Drag Timer 1.3.0 adds named multi-timer routines, richer Quick start presets, configurable menu-bar presentations, local history, and explicit handling for simultaneous expiries.

## Changes

- Save, edit, duplicate, reorder, and launch named routines such as Pomodoro, Morning routine, or Workout.
- Give each Quick start preset its own label, duration, color, symbol, sound, volume, loop, notification, and snooze settings.
- Show the nearest deadline, active count, a pinned timer, or a progress ring in the menu bar.
- Resolve each expiry with Snooze, Restart, Silence, or Mark done, including when several timers expire together.
- Review locally stored completion/cancellation history and restart a historical timer.
- Check GitHub Releases manually or once daily without automatic installation.
- Fix the release checksum file so it verifies correctly after downloading both assets.

## Compatibility

- Requires macOS 14 or newer.
- The v1.3.0 public binary is Apple Silicon (`arm64`) only. Universal Apple Silicon and Intel packaging begins with v1.3.1.
- Existing v1.2.0 timers and settings migrate forward. Presets and routine timer snapshots remain independent after creation.

## Installation

1. Download `Drag-Timer-1.3.0-macos.zip` and `SHA256SUMS.txt` from the GitHub release.
2. Verify the checksum, unzip the archive, and move `Drag Timer.app` to Applications.
3. Control-click the app and choose **Open**, or use **Open Anyway** in System Settings > Privacy & Security.

## Verification

Run `shasum -a 256 -c SHA256SUMS.txt` in the directory containing both downloaded assets.

## Known limitations

The app is ad-hoc signed, not Developer ID signed or notarized. macOS therefore requires a one-time Gatekeeper confirmation. Intel Macs cannot run the v1.3.0 artifact.
